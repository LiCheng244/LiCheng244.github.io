iOS开发 - 数据存储 NSUserDefaults 的使用

发表于 2016-11-03   |   分类于 iOS__03【 数据存储 】   |     |   阅读次数

简介:
NSUserDefaults是一个单例，在整个程序中只有一个实例对象，他可以用于数据的永久保存，而且简单实用，这是它可以让数据自由传递的一个前提。

NSUserDefaults适合 存储轻量级 的本地数据 ，比如要保存一个登陆界面的数据，用户名、密码之类的，个人觉得使用NSUserDefaults是首选。下次再登陆的时候就可以直接从NSUserDefaults里面读取上次登陆的信息。就像读字符串一样，直接读取就可以了。

iOS开发 - 数据存储 CoreData 的使用

发表于 2016-11-03   |   分类于 iOS__03【 数据存储 】   |     |   阅读次数

简介:
CoreData 是数据持久化存储的最佳方式.
CoreData 是基于 sqlite 的封装, 数据保存到一个数据库文件中(表).
iOS5 以后才出现.
它提供了对象-关系映射(ORM)的功能, 既能够将oc对象转化成数据保存在sqlite文件中, 也能将保存在数据库中的数据还原成oc对象.
数据操作期间, 我们不需要编写任何的sql语句.
数据最终的存储类型可以是：SQLite数据库，XML，二进制，内存里，或自定义数据类型.
好处：能够合理管理内存,避免使用sql的麻烦, 更高效. 不需要我们手动去创建model类, 可以通过一些操作, 制动生成.

iOS开发 - HTTP 的请求头和响应头

发表于 2016-11-01   |   分类于 iOS__04【 网络相关 】   |     |   阅读次数

关于HTTP消息头:

HTTP消息头是在，客户端请求（Request）或服务器响应（Response）时传递的，位请求或响应的第一行，HTTP消息体（请求或响应的内容）是其后传输。
HTTP消息头，以明文的字符串格式传送，是以冒号分隔的键/值对，如：Accept-Charset: utf-8，每一个消息头最后以回车符(CR)和换行符(LF)结尾。HTTP消息头结束后，会用一个空白的字段来标识，这样就会出现两个连续的CR-LF。

HTTP消息头由IANA（The Internet Assigned Numbers Authority，互联网数字分配机构）来整理和维护。其标准最早来源于RFC 4229。IANA将其整理到了消息头文档，文档中还包括了一些新提出的信息头。

HTTP消息头支持自定义， 自定义的专用消息头一般会添加’X-‘前缀。
